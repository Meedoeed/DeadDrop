# DeadDrop - Секретный сервис на Go

## О проекте

**DeadDrop** - это учебный проект, созданный в рамках цикла статей "Ботаем net/http на Go". Это сервис для безопасной передачи конфиденциальной информации, где сообщения самоуничтожаются после прочтения. Проект разрабатывается поэтапно, каждая версия представляет собой законченный этап разработки.

## Версии проекта

### v1: Фундамент
- **Соответствует:** Части 1 цикла статей
- **Статус:** Завершена
- **Основные темы:**
  - Основы net/http, запуск сервера
  - Обработчики и маршрутизация
  - Middleware (логирование, recovery)
  - HTML-шаблоны и layout'ы
  - Статические файлы и embed FS
  - Базовая архитектура проекта

### v2: Обработка данных
- **Соответствует:** Части 2 цикла статей
- **Статус:** Завершена
- **Основные темы:**
  - Обработка POST-запросов и форм
  - Загрузка файлов на сервер
  - In-memory хранилище с TTL
  - Динамические маршруты (/secret/{id})
  - Валидация пользовательского ввода
  - Полный цикл создания/просмотра секретов

### v3: Безопасность
- **Соответствует:** Части 3 цикла статей
- **Статус:** В разработке
- **Планируемые темы:**
  - CSRF-токены и security headers
  - Хеширование паролей
  - Rate limiting
  - HTTPS настройка
  - Защищенные cookie

### v4: Production-готовность
- **Соответствует:** Части 4 цикла статей
- **Статус:** Планируется
- **Планируемые темы:**
  - Graceful shutdown
  - Structured logging (slog)
  - Конфигурация через переменные окружения
  - Health checks и метрики

### v5: Масштабирование и деплой
- **Соответствует:** Части 5 цикла статей
- **Статус:** Планируется
- **Планируемые темы:**
  - База данных (PostgreSQL/SQLite)
  - Кэширование (Redis)
  - Фоновые задачи
  - Docker и Docker Compose
  - CI/CD пайплайн

## Быстрый старт

### Предварительные требования
- Go 1.21 или выше
- Git

### Запуск любой версии

```bash
# Клонировать репозиторий
git clone https://github.com/Meedoeed/DeadDrop.git
cd DeadDrop

# Перейти в нужную версию
cd v1  # или v2, v3, etc.

# Запустить сервер
make run
# Или напрямую
go run cmd/api/main.go
```

Сервер будет доступен по адресу: http://localhost:8080

## Технологический стек

- **Язык:** Go 1.21+
- **Веб-сервер:** net/http (стандартная библиотека)
- **Шаблоны:** html/template
- **Хранилище:** In-memory (v1-v2), планируется PostgreSQL (v5+)
- **Статика:** embed.FS
- **Инструменты:** Make, Air (hot reload)

## Для начинающих

Этот проект идеально подходит для изучения веб-разработки на Go:

1. **Начните с v1** - фундаментальные концепции
2. **Переходите к v2** - обработка данных

4. **Пишите код вместе со статьями** для лучшего понимания

## Функциональность

### Реализовано в v2:
- Создание секретных сообщений
- Загрузка файлов (до 10MB)
- Генерация уникальных ссылок
- Защита паролем
- Автоматическое удаление после прочтения
- In-memory хранилище с TTL

### В разработке:
- Постоянное хранилище (БД)
- Сессии пользователей
- Уведомления по email
- API для мобильных приложений
- Аудитория сообщений
